# Retraction Log

> 本文件記錄每次需要「收回」判斷的情況，供自我反思與改進。
> Brooks 是 single source of truth。在資訊足夠的情況下出現收回，代表推理流程有問題，不是知識不足。
> 每次收回後，AI 必須在此記錄根因，並提煉出可防止同類錯誤的規則。

---

## 收回記錄格式

```
### YYYY-MM-DD｜[簡短描述]
**原始判斷**：
**收回原因**：
**根因分類**：資訊不足 / 推理流程錯誤 / 概念誤用
**根因說明**：
**防止規則**：
```

---

## 記錄

### 2026-04-26｜對比論證未雙向驗證

**原始判斷**：
Session 003 的 wedge top 做空是「跟 macro/AIS 走」的高品質交易；本次 wedge bottom 做多是「逆 AIS」的低品質交易；兩者不對稱，不能類比。

**收回原因**：
使用者追問「session 003 三推的當下 AIS 是 Long 嗎？」，確認後發現：session 003 在三推期間 AIS 確實是 Long（AIL），wedge top 是 AIS 從 Long 翻 Short 的翻轉點，進場時跟新的 AIS 同方向。這個結構和本次 wedge bottom 的邏輯完全對稱，「不對稱」的結論不成立。

**根因分類**：推理流程錯誤

**根因說明**：
先有結論（兩者不對稱），再往回找支撐論點。在做對比論證時，只對其中一個情境走了完整驗證，沒有對另一個情境用同樣的 framework 走一遍，就下了結論。結論聽起來合理（因為論點本身不是假的），但沒有完整驗證對稱性。

**防止規則**：
做任何「A 和 B 的對比」論證前，必須先用同一個 framework 對 A 和 B 各自完整走一遍，列出結果後再說結論。不能只走一邊就下結論。

具體 checklist（對比論證時強制執行）：
```
情境 A：三推期間 AIS = ? → 進場時 AIS = ? → 進場跟 AIS 同向？
情境 B：三推期間 AIS = ? → 進場時 AIS = ? → 進場跟 AIS 同向？
→ 兩邊列完，再說對不對稱
```
